# אפיון ממשק משתמש — אפליקציית ספירת מלאי תחמושת

## 1. עקרונות עיצוב כלליים

### שפה חזותית
- **כיוון**: RTL מלא, עברית
- **פלטת צבעים**: ירוק צבאי (#4a5d23) כצבע ראשי, רקע בהיר (#f5f5f0), כרטיסים לבנים
- **טיפוגרפיה**: Segoe UI / Tahoma / Arial — פונטים סיסטמיים בלבד, בלי טעינה חיצונית
- **פינות מעוגלות**: 10px לכרטיסים, 6px לשדות קלט
- **צללים**: עדינים (0 2px 12px rgba(0,0,0,0.08)) ליצירת עומק

### עקרונות UX
- **Mobile-first**: מותאם למסכי 360px+, touch targets מינימום 44px
- **מינימליזם**: כל מסך מציג רק מה שרלוונטי לפעולה הנוכחית
- **הזנה מהירה בשטח**: מינימום הקלדה, מקסימום בחירה מרשימות
- **משוב מיידי**: טוטאלים מתעדכנים חי, אנימציות עדינות, הודעות שמירה
- **שמירה על הקשר**: המשתמש תמיד יודע איפה הוא (יחידה, לשונית, רמה)

## 3. מסך כניסה (Session Setup)

### מטרה
הגדרת פרטי הדוח לפני תחילת ההזנה. זהו למעשה "Report Session Setup".

### Layout
- ממורכז במסך, רוחב מקסימלי 480px
- לוגו + כותרת בראש
- כרטיס טופס יחיד
- מתחתיו — רשימת טיוטות שמורות (אם קיימות)

### שדות הטופס

| שדה | סוג | התנהגות |
|-----|------|---------|
| מספר גדוד | text input | ברירת מחדל "334", נזכר מהפעם הקודמת |
| תאריך ושעה | datetime-local | אוטומטי (עכשיו), ניתן לשינוי |
| שם מדווח | text input | ריק, נזכר מהפעם הקודמת (localStorage) |
| רמת משתמש | segmented control (3 כפתורים) | צוות / פלגה / סוללה. נזכר מהפעם הקודמת |
| שם יחידה | text input | placeholder דינמי לפי רמה שנבחרה |
| מספר צוותים | counter (+/−) | מופיע רק עבור פלגה/סוללה. טווח 1-10 |

### Segmented Control — רמת משתמש
- שלושה כפתורים צמודים בשורה אחת
- הכפתור הפעיל: רקע ירוק + טקסט לבן
- הכפתורים הלא פעילים: רקע לבן + טקסט ירוק
- גבול ירוק חיצוני מסביב לכל הקבוצה

### Counter — מספר צוותים
- כפתור "−" | מספר | כפתור "+"
- מתחתיו: badges קטנים עם שמות הצוותים האוטומטיים ("צוות א'", "צוות ב'", ...)

### כפתור "התחל ספירה"
- רוחב מלא
- רקע ירוק gradient
- טקסט לבן, bold
- ולידציה: שם מדווח ושם יחידה חובה

### אזור טיוטות שמורות
- מופיע רק אם יש טיוטות ב-IndexedDB
- כותרת "טיוטות שמורות" עם קו תחתון
- כל טיוטה: כרטיס עם שם יחידה + רמה + מדווח + תאריך עדכון
- שני כפתורים: "טען" | "מחק" (עם confirm)

## 4. App Header (בר עליון)

### מוצג בכל המסכים חוץ ממסך הכניסה

### Layout
- רקע ירוק gradient, sticky top, z-index 100
- צד ימין: מידע — שם יחידה (bold), רמה (badge), מספר גדוד
- צד שמאל: כפתורים עגולים — בית (🏠), מחק (🗑️)

### אינטראקציות
- 🏠 → חזרה למסך כניסה (בלי מחיקת session)
- 🗑️ → confirm dialog → ניקוי הכל וחזרה למסך כניסה

### 5.1 Main Tab Bar

#### לפי רמת צוות:
| לשונית | תוכן |
|---------|-------|
| בטן | טופס חימושים |
| חוץ | טופס חימושים |

#### לפי רמת פלגה:
| לשונית | תוכן |
|---------|-------|
| צוות א' | sub-tabs: בטן / חוץ |
| צוות ב' | sub-tabs: בטן / חוץ |
| ... | ... |
| אלפ"א | טופס חימושים ישיר |

#### לפי רמת סוללה:
| לשונית | תוכן |
|---------|-------|
| צוות א' | sub-tabs: בטן / חוץ |
| צוות ב' | sub-tabs: בטן / חוץ |
| ... | ... |
| אלפ"א | טופס חימושים ישיר |
| רמסע | טופס חימושים ישיר |

#### סגנון
- scrollable אופקית (overflow-x: auto)
- לשונית פעילה: טקסט ירוק + קו תחתון ירוק 3px
- לשוניות לא פעילות: טקסט אפור
- רקע לבן

### 5.2 Sub Tab Bar
- מופיע רק כשהלשונית הראשית היא צוות (בפלגה/סוללה)
- שני כפתורים: "בטן" | "חוץ"
- סגנון segmented: רקע ירוק בהיר, כפתור פעיל לבן עם צל

### 5.3 טופס חימושים (Ammo Form)

#### מבנה כללי
- 5 קבוצות (accordion cards): פגז, חנ"ה, מרעום, תחלים, קליעית
- כל קבוצה היא כרטיס עם header לחיץ (collapse/expand)
- padding-bottom: 90px (כדי שה-bottom bar לא יחסום)

#### Header של קבוצה
```
◀/▼  פגז                          42  3 פריטים
[icon] [שם הקבוצה]        [badge טוטאל] [מונה שורות]
```
- לחיצה → toggle open/close
- ◀ כשסגור, ▼ כשפתוח (RTL)
- Badge טוטאל: עגול ירוק, מופיע רק אם > 0
- מונה פריטים: טקסט אפור

#### קבוצת "תחלים" (quantity-only)
- אין שורות חימוש
- שדה כמות בודד: `כמות: [____]`
- input type=number, text-align: center, רוחב 120px

#### שורת חימוש (לקבוצות עם דגמים)

**Desktop/Tablet (>480px):**
```
┌──────────────────────┬──────────────┬──────────┬────┐
│ דגם (dropdown)       │ סדרה (אם חנ"ה)│ כמות     │ ✕  │
└──────────────────────┴──────────────┴──────────┴────┘
```
- Header row מעל: "דגם", "סדרה" (אם רלוונטי), "כמות"

**Mobile (<480px):**
- דגם: רוחב מלא
- סדרה + כמות: שורה שנייה, 50%-50%
- כפתור X קטן מימין

#### פרטי שדות

| שדה | סוג | פרופורציה | הערות |
|-----|------|-----------|-------|
| דגם | select/dropdown | flex: 3 | placeholder: "בחר דגם..." |
| סדרה | text input | flex: 2 | רק עבור חנ"ה. placeholder: "סדרה" |
| כמות | number input | flex: 1.5, min-width: 70px | text-align: center, inputMode: numeric |
| הסרה | button (✕) | 36x36px fixed | עגול, hover אדום בהיר |

#### כפתור "+ הוסף דגם"
- רוחב מלא
- גבול מקווקו (dashed) 2px
- טקסט ירוק, hover: רקע ירוק בהיר + גבול ירוק מלא
- הטקסט דינמי: "+ הוסף דגם פגז", "+ הוסף דגם חנ"ה", וכו'

#### אנימציה
- שורה חדשה: fadeIn 0.2s (opacity + translateY)

### 5.4 Bottom Actions Bar

- מקובע בתחתית המסך (position: fixed, bottom: 0)
- רקע לבן + צל עליון + border-top
- שני כפתורים בשורה אחת:

| כפתור | סגנון | פעולה |
|--------|-------|-------|
| 💾 שמור טיוטה | outlined (גבול ירוק, רקע לבן) | שמירה ל-IndexedDB + toast |
| 📊 סיכום ויצוא | filled (רקע ירוק, טקסט לבן) | מעבר למסך סיכום |

- לאחר שמירה מוצלחת: הכפתור הופך לרגע לירוק עם "✓ הטיוטה נשמרה" (2 שניות)

## 7. Toast Notifications

- מיקום: fixed, bottom 80px (מעל ה-bottom bar), מרכז אופקי

## 9. Responsive Breakpoints

| Breakpoint | התאמות |
|------------|--------|
| > 480px | layout רגיל, headers בטבלאות, שדות בשורה |
| ≤ 480px | header טבלה מוסתר, שדות ב-wrap, כפתורי יצוא בעמודה, report info בעמודה, טבלאות font קטן יותר |

---

## 10. Print / PDF Styles

בעת הדפסה (`@media print`):
- מוסתרים: App Header, Bottom Actions, כפתורי יצוא, כפתור חזור
- summary-container: ללא padding, רוחב מלא
- טבלאות: ללא צל, font קטן יותר, `page-break-inside: avoid`
- כרטיס סה"כ: רקע אפור, טקסט שחור (לא gradient)

## 11. מצבים מיוחדים

### אין טיוטות שמורות
- אזור הטיוטות לא מוצג כלל (לא placeholder ריק)

### Session ריק (אין נתונים)
- כל הקבוצות פתוחות כברירת מחדל
- badge טוטאל לא מוצג (רק כש-total > 0)
- מסך סיכום מציג טבלאות עם אפסים

### שגיאות ולידציה
- שם מדווח ריק → `alert("נא להזין שם מדווח")`
- שם יחידה ריק → `alert("נא להזין שם יחידה")`
- כמויות: מינימום 0, אין ולידציה מתקדמת ב-MVP

## 12. אינטראקציות מרכזיות — סיכום

| פעולה | תוצאה |
|--------|-------|
| בחירת רמה | עדכון placeholder של שם יחידה + הצגת/הסתרת counter צוותים |
| שינוי מספר צוותים | עדכון badges שמות צוותים |
| התחל ספירה | יצירת session + מעבר למסך הזנה |
| לחיצה על header קבוצה | toggle collapse/expand |
| + הוסף דגם | שורה חדשה עם fadeIn |
| שינוי dropdown דגם | עדכון modelId + modelName |
| שינוי כמות | עדכון מיידי, badge טוטאל מתעדכן |
| ✕ הסרת שורה | שורה נמחקת |
| שמירת טיוטה | IndexedDB save + toast |
| סיכום ויצוא | מעבר למסך סיכום עם report נורמליזציה |
| יצוא Excel | הורדת קובץ .xlsx |
| הדפסה/PDF | `window.print()` |
| טעינת טיוטה | session מוחלף + מעבר למסך הזנה |
| מחיקת טיוטה | confirm → מחיקה מ-IndexedDB |
| 🏠 | חזרה למסך כניסה (session נשמר ב-memory) |
| 🗑️ | confirm → ניקוי session + חזרה |
